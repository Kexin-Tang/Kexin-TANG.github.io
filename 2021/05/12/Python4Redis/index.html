本文主要介绍基于Python来连接、操作Redis数据库。在Python中，使用redispy库来进行连接。关于redispy详细的文档可以参考GitHub。

连接Redis

在Python3中返回的是bytes类型，想要直接获取str类型，需要设置decode_responses=True

注意，在redispy中是没有设置SELECT操作更换数据库id，所以请为每个数据库都创建独立的连接实例。

在默认情况下，redispy会自动给每个连接实例创建对应的连接池，但是用户也可以手动将连接池实例放入已存在的连接池中。

Transaction

在redispy中，可以使用pipeline实现transaction

存在一种更便捷的方法，用于处理try-catch和加锁等操作。它需要一个回调函数，一个管道对象和任意数量的键都将被监视

数据类型

对于基本的Redis操作都有支持。

  • 对于最普通的，传入参数都是func(key: str, val: str/list/tuple)
  • 对于多个设置，如mset等，传入的都是键值对的mapping
  • 对于设置时间相关的，如setex等，func(key: str, time: int, value: str)
  • 对于有范围的，如lrange等，传入的都是func(key: str, start: int, end: int)
  • 对于有个数的，如lrem等，传入的参数func(key: str, num: int, value: str)
  • 对于有间隔的，如incrby等，传入的参数func(key: str, step: int)

注意，对于list/tuple等，传入时参数要为*args；对于dict，传入时参数要为**kwargs

PubSub

订阅

订阅非常简单，只需要输入以下的指令创建pubsub实例，然后订阅指定的频道即可，其中psubscribe可以匹配正则表达式。

通过get_message可以获得实例对应的信息。其中消息体的具体含义：

  • type – subscribe, unsubscribe, psubscribe, punsubscribe, message, pmessage中的某一个
  • channel – 订阅的频道或者是有新消息的频道
  • pattern – 与已发布消息的频道匹配的模式。 除pmessage类型外，在所有情况下均为None
  • data – 消息信息。如果是有消息，则为消息内容，否则为频道id

发布

使用p.publish进行消息发布，可以指定发布的位置和消息内容。

取消订阅

对于subscribe，使用unsubscribe；对于psubscribe，使用punsubscribe。其可以指定取消订阅的频道名字/模式串，如果不传入任何参数则视为对所有频道取消订阅。

通过回调函数

redis-py支持使用回调函数实现pubsub。回调函数要传入一个message变量，该变量为dict类型，就类似于之前的get_message返回的dict。如果想要订阅一个频道，则需要传入一个dict，其key为channelName，value为回调函数。

当一个信息被读取后，handler会自动帮我们在message这个dict中保存信息，并调用回调函数，所以get_message返回的都是None而非信息。

只接收新消息

通过设置p = r.pubsub(ignore_subscribe_messages=True)，这样在p.get_message()时，如果没有新消息，则不会返回任何值；如果有信息消息，才会返回值。

读取消息

轮询

get_message内部的实现机制是快速的调用select，在多个数据库之间遍历，寻找是否有新的消息，如果有则获取，否则返回None。

监听

listen()是一个生成器，其会阻塞其他进程直到有消息。

另起线程

可以将事件的循环放入另一个独立的线程中。run_in_thread()可以创建一个独立的线程，并在该线程内部开启一个循环，不断使用第一种方法查看是否有可用的信息。

注意：要使用该方法，必须使用回调函数。

使用Lua脚本

redis-py支持 EVAL, EVALSHA, 和 SCRIPT 这些操作，但是为了让代码更具有Python的风格，同时省去一些不必要的麻烦，我们可以使用独有的方法来加载并执行Lua脚本。

在执行lua_script时，还可以设置client，用于表明脚本在哪个平台上运行。如果不指明，则默认为注册lua_script的平台。

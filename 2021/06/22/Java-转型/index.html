转型

转型一般分为两种情况：

  • 向上转型：子类 → 父类（可以是类/接口），Father f = new Son()
  • 向下转型：父类 → 子类，Son s = (Son) f

如何判断能否转型？只要右边的能当做左边的使用就是正确的，或者说只要左边有的功能右边都有就是正确的。比如手机 = 苹果手机，苹果手机能当成普通手机使用，那么这样的转换就是合理的。

向上转型

在向上转型时：

  1. 父类的引用所指向的属性是父类的属性。
  2. 如果子类重写了父类的方法，那么父类引用调用的方法是子类的方法,这个叫动态绑定。如果子类没有重写父类方法，那么调用的还是父类方法。
  3. 不能调用子类独有的方法。

如果非要调用子类的属性，那么需要在父类定义一个属性getter，然后在子类中进行重写

向下转型

只有向上转型的引用才能安全的使用向下转型，如果父类引用指向父类本身，那么使用向下转型可能会出错。

总结

向上转型会丢失子类特有的方法，但是仍然能保持子类中重写的方法；为了获取子类特有的方法，可以在向上转型基础上增加向下转型即可。

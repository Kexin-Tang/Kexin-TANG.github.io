泛型

概念

即预定义一个数据结构，内部可以存储统一的元素类型，但是此时未知数据类型。因此我们可以使用泛型，假设其为泛型类型 E。然后在初始化时再指定具体的泛型类型。

注意：如果ADHero和APHero是Hero的子类，那么ArrayList<Hero>可以储存Hero、ADHero和APHero类型的元素，该原则符合多态的思想。

为什么不使用Object呢？毕竟Object是所有类型的父类。

由于Object是父类，那么如果所有元素都用Object接收，相当于多态，那么使用元素特有方法的时候就需要向下转型，非常麻烦。

定义泛型

泛型的 Class , Method 和 Interface 都是使用类似的思想。

泛型通配符

假设我们定义了一个ArrayList，并准备设计一个函数，以该ArrayList为参数，那么问题来了：形参应该定义成什么类型呢？用户可能传递一个ArrayList<String>，也可能是一个ArrayList<Integer>，程序员不可能给每一种类型都设计一个功能一样的函数。因此，使用泛型通配符就可以解决该问题。

泛型通配符使用<?>的形式表明接收的泛型类型未知。

注意：通配符只能在接收时使用，不能实例化一个含有通配符的泛型，即ArrayList<?> xx = new ArrayList<>();是不允许的。

集合框架

有<>的表示为 “接口Interface”，否则为 “类Class”。

  • List的特点：
    • 存储时按序存储
    • 可以存储重复元素
    • 可以迭代
    • 有索引
  • Set的特点：
    • LinkedHash是有序的；否则是无序的
    • 不可以存储重复元素
    • 无法使用for循环迭代
    • 没有索引

Collection共性方法

  • boolean add(E e) – 添加元素到集合
  • boolean remove(E e) – 删除指定的元素
  • void clear() – 清空集合
  • boolean contains(E e) – 是否存在元素
  • boolean isEmpty() – 判断集合是否为空
  • int size() – 元素个数
  • Object[] toArray() – 转为普通数组

迭代器 Iterator

迭代器是 Collection 集合中取出元素的通用方法。每次先判断是否还有元素，如果是，那么继续取出下一个；如果不是，那么结束迭代。

  • boolean hasNext() – 判断是否还有元素
  • E next() – 返回下一个元素

注意：由于迭代器是一个接口，所以无法实例化，但是在 Collection 中有一个迭代器实现类，因此我们可以使用一个迭代器接口变量去接收该方法返回的迭代器实现类(多态)。

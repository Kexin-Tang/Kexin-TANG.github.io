集合框架

集合框架如上图所示，可以分为两个大类：存储元素集合的Collection；存储key-value键值对的Map。这两个大类又分别有一些具体的实现类。除此之外，还有一个Collections工具库，专门实现了一些能够操控、处理集合的方法，比如常见的“排序”，“倒置”等。

  • List的特点：
    • 存储时按序存储
    • 可以存储重复元素
    • 可以迭代
    • 有索引
  • Set的特点：
    • LinkedHash是有序的；否则是无序的
    • 不可以存储重复元素
    • 无法使用for循环迭代
    • 没有索引

迭代器 Iterator

迭代器是 Collection 集合中取出元素的通用方法。每次先判断是否还有元素，如果是，那么继续取出下一个；如果不是，那么结束迭代。

  • boolean hasNext() – 判断是否还有元素
  • E next() – 返回下一个元素

注意：由于迭代器是一个接口，所以无法实例化，但是在 Collection 中有一个迭代器实现类，因此我们可以使用一个迭代器接口变量去接收该方法返回的迭代器实现类(多态)。

List

List的数据结构都有一系列相同的操作：

  • boolean add(E e) / boolean add(int idx, E e) – 添加元素到集合
  • E get(int idx) – 获取元素
  • E set(int idx, E e) – 替换指定位置的元素，并返回旧的元素
  • boolean remove(E e) / boolean remove(int idx) – 删除指定的元素
  • boolean contains(E e) – 是否存在元素
  • boolean isEmpty() – 判断集合是否为空
  • int size() – 元素个数
  • Object[] toArray() – 转为普通数组
  • String toString() – 转为字符串

ArrayList

底层其实还是数组类型。当增加或删除元素时，会先根据当前的数组长度计算出更新后的数组长度，然后通过Arrays.copy()进行内容的复制，实现增删。

对于查询，可以通过下标直接访问。

因此，ArrayList的更新效率低，但是查询效率高。

更多内容，详见Java基础。

LinkedList

底层是双向链表结构。根据链表的特性可知，其更新效率高，查询效率低。

由于其增删效率很高，因此常见的栈、队列等数据结构都可以使用链表来模拟。

  • 增
    • 在链表头部增加 – addFirst(E e)
    • 在链表尾部增加 – add(E e) / addLast(E e)
  • 删
    • 删除头部元素 – removeFirst()
    • 删除尾部元素 – removeLast()
  • 查
    • 查询头部元素 – getFirst()
    • 查询尾部元素 – getLast()

虽然链表不能直接根据下标访问，但是java中重写了linkedlist的一些方法，所以仍然可以使用类似set(int index, E e)的方法，通过指定下标来直接访问指定位置。

Vector

是比较早期的概念，现已经被ArrayList所替代，基本不再使用。

与ArrayList相比，他们有很多相同点：

  • 可以随机访问
  • 有序存放
  • 可边长
  • 支持泛型
  • 可存储重复元素

但是他们也有很多不同点：

  • Vector是同步的，这导致其在多线程环境下效率低
  • Vector不支持迭代器，需要使用独有的方法去遍历
  • Vector有很多独有的方法，增加了复杂性

Stack

是基于Vector实现的栈数据结构，现在很少直接使用该包。

Set

HashSet

HashSet的底层是：数组+链表/红黑树。具体而言，当存入一个元素时，会使用方法hashCode()计算其hash值，然后将该元素放入数组中hash值对应的下标处。如果该下标处没有元素，那么直接放入；如果已经有元素了，那么会像链表一样在之前已有的后面添加节点。

由于相同的元素会有相同的hash值，不同的元素可能也会凑巧有相同的hash值，因此Set在通过hash值找到对应位置时，还会依次调用方法equals()比较要放入的元素和已经存储的元素是否相同，如果有相同的，则终止操作，实现去重，否则就继续放入。

**注意：如果想使用HashSet存储自定义的类，需要在类中重写hashCode()和equals()**。

常用的API：

  • add(E e)，添加一个元素，由于Set是不管放入顺序的，所以不能指定放入的index
  • remove(E e)，删除一个元素，同样由于没有下标，所以不能指定删除哪个位置的，只能删除某个元素
  • contains(E e)，查看是否包含某个元素
  • size()，返回set内的元素个数

由于没有下标，所以不能使用常规的for循环，只能使用增强for循环或者迭代器进行遍历。

LinkedHashSet

与 HashSet 基本一致，只不过底层是哈希表+链表，即该 HashSet 可以做到记录存储的顺序。

Map<K, V>

以键值对的形式存储内容，和 Python 中的字典类似。

HashMap<K, V>

底层是哈希表，无序、键值要求不能重复、键和值可以使用 null。

**注意：同 Set 一样，由于 key 不能重复，所以如果我们要用自定义的类作为 key，就必须在类中重写 equals 和 hashCode 方法**。

常用API：

  • public V put(K key, V value)：如果不存在该key，则新建，并返回null；如果存在该key，则替换，并返回被替换的value
  • public V remove(Object key)
  • public V get(Object key)
  • boolean containsKey(Object key)
  • public Set<K> keySet()
  • public Set<Map.Entry<K, V>> entrySet()
  • public Collection<V> values()

Entry内部类

每次初始化Map时，都会生成一个Entry静态内部类，存储了每一对键值对的内容。使用 entrySet() 方法后会生成一个由 Map.Entry<K, V> 结构构成的 Set，然后通过遍历该 Set 即可遍历到每一对键值对。再通过 getKey() 和 getValue() 方法分别获取键值对中的键和值。

Collections工具库

该库位于 java.util.Collections，主要是提供了许多操控集合的方法。

  • public static <T> boolean addAll(Collection<T> c, T..., elements)：向集合中一次性添加多个元素
  • public static <T> void sort(List<T> list)：按照默认规则排序
  • public static <T> void sort(List<T> list, Comparator<? super T>)：按照指定的规则排序

Comparable接口

Comparable 是排序接口，如果一个类实现了排序接口，就意味着该类“支持默认排序规则”，可以使用 Collections.sort() 按照默认的排序规则进行排序。

具体而言，**我们要在用户定义的类中实现 Comparable，并重写 compareTo() 方法**。

默认情况下res = x.compareTo(y)：如果 res<0* 则代表 *x<y* (或者说x排在y左侧)；如果 *res=0* 则代表 *x=y*；如果 *res>0 则代表 x>y (或者说x排在y右侧)。

综上，**如果想要定义默认排序规则为升序，则在 compareTo() 中返回 this-o，否则返回 o-this**。

Comparator接口

comparator 是比较器接口，我们可以通过实现该接口来新建一个比较器。

具体而言，**我们要实现 Comparator，并重写 compare() 方法，然后将该比较器放入 Collections.sort() 的第二个参数作为排序规则**。

默认情况下res = compare(Object o1, Object o2)：如果 res<0* 则代表 *o1<o2* (o1排在o2左侧)；如果 *res=0* 则代表 *o1=o2*；如果 *res>0 则代表 o1>o2（o1排在o2右侧）

与上述的 Comparable 类似，返回 o1-o2 则为升序，否则为降序。

Comparable VS Comparator

一个定义的是默认排序方法，使用sort时不需要额外参数；一个定义的是自定义排序方法，使用sort时需要传入自定义方法。这就相当于 Comparable 是内部（隐式）排序方法， Comparator 是外部（显式）排序方法。

of

在JDK9中，为 List接口，Set接口 和 Map接口 增加了一个静态方法 – of，用于在初始化时一次性添加多个元素。

注意：

  1. of只适用于这三个接口，不适用于其实现类，如ArrayList
  2. 返回的集合是不可改变的，即不能使用诸如 add, remove, set等
  3. Set 和 Map 使用 of 时不能传入相同的元素，否则会抛出异常
